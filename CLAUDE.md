# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

The live website for studiochenille.com — a luxury interior design firm in Ascot, UK. Hand-written static HTML/CSS/vanilla JS. **No build system, no framework, no package.json, no test suite, zero runtime dependencies** — keep it that way; mobile speed matters (Google Ads land on these pages).

## Commands

```bash
python3 -m http.server 8000        # local preview — no build step
bash tools/hero-video-to-frames.sh hero.mp4   # convert hero video → scroll-scrub frames
bash tools/encode-video-tiers.sh   # 4K masters → all device-tier encodes + posters
```

Deploy = `git push origin main` (GitHub Pages, `CNAME` → studiochenille.com). `.htaccess` is for Apache hosting and is ignored by Pages. Pages caches assets ~10 min: `index.html` references its CSS/JS with `?v=N` query strings — bump N whenever those files change, or stale-HTML/fresh-JS mixes cause breakage (a stuck dark preloader overlay was one such bug; main.js keeps a guard that removes orphaned `.preloader` nodes from stale HTML). Verification is manual: serve locally and check in a browser (Playwright chromium is cached on this machine under `~/.cache/ms-playwright/*/chrome-linux64/chrome` if headless screenshots are needed).

## Architecture

### Shared markup is hand-duplicated

There is no templating. Header, footer, nav, cookie banner, and JSON-LD schema are copy-pasted across ~40 pages. Any site-wide change (nav item, footer link, phone number, company number 17164468) must be applied to **every** HTML file including `blog/*.html` — grep first to find all occurrences, and check with `grep -rL` afterwards that none were missed.

### Page types

- `index.html` — homepage with the scroll-scrubbed hero journey (see below)
- Service pages: `residential-interior-design.html`, `hotel-hospitality-design.html`, `commercial-interior-design.html`, `show-home-design.html`
- ~20 local-SEO pages `interior-design-<town>.html` in two tiers: full (~16KB, e.g. Ascot, Windsor) and lighter satellite (~11KB, e.g. Chelsea, Esher)
- `blog/*.html` articles, plus `faq.html`, `about.html`, `contact.html`, `areas-we-serve.html`

New pages must be added to `sitemap.xml`. Every page carries meta/OG tags and JSON-LD (`InteriorDesignBusiness` schema). `llms.txt` describes the site for AI crawlers.

### CSS/JS layout

- `css/style.css` — global design system: CSS custom properties, warm luxury palette (cream/gold/charcoal), Playfair Display headings + Inter body
- `js/main.js` — preloader (homepage only), header scroll state, mobile nav, testimonial slider, contact form (posts to FormSubmit), cookie consent
- `css/effects.css` + `js/effects.js` — scroll-progress bar and decorative effects, all rAF-throttled
- `css/hero-journey.css` + `js/hero-journey.js` — homepage hero engine

### Hero journey (homepage)

A 320vh wrapper (`#heroJourney`) pins a sticky 100svh hero; scroll progress drives a lerp-smoothed timeline. Two auto-selected engines:

- **zoom mode** (default): layered zoom-through of three portfolio photos
- **video mode**: canvas frame-sequence scrub; activates when `images/hero/manifest.json` has `frames > 0`. Frames are generated by `tools/hero-video-to-frames.sh` from a Seedance 2.0 video — the generation recipe is `HIGGSFIELD-HERO-RECIPE.md`. Delete the manifest values back to the placeholder (`frames: 0`) to revert to zoom mode.
- **videoscrub sub-mode**: landscape screens whose physical width (`innerWidth × devicePixelRatio`, DPR capped at 2) exceeds a manifest tier's `min_phys` scrub a dense-keyframe mp4 (`videos/hero/hero-desktop-1440/2160.mp4`, g=6 so any `currentTime` seek lands ≤3 frames from an IDR) instead of the 1280px webp frames; seeks are chained one-in-flight, and any error/8s stall falls back to the webp frame path. Retina phones get the `mobile.hd` portrait frame set (`frames-mobile-v4`, 900×1600). Poster `<picture>` uses `srcset` w/x descriptors so each viewport preloads the poster cut from its own tier's footage.

Falls back to a classic static hero when JS is absent, `prefers-reduced-motion` is set, or `overflow-x: clip` is unsupported.

All tier encodes are cut from the Topaz-4×-upscaled 4K masters in
`~/studiochenille-masters/4k/` (55MB+ HEVC each — kept out of this public
repo; raw 720p Seedance originals in `~/studiochenille-masters/`) by
`tools/encode-video-tiers.sh`. Never "sharpen" a shipped clip by re-encoding
it — go back to the master.

**Traps:**
- `body` must use `overflow-x: clip`, never `hidden` — `hidden` makes body a scroll container and silently breaks the sticky pinning (style.css declares `hidden` then `clip` deliberately, as a fallback chain).
- The header's transparent→solid switch in `main.js` is coupled to `#heroJourney`'s height; changing the wrapper height or hero structure affects it.

### Portfolio ambient videos (homepage Featured Projects)

Each grid tile in `#projects` carries `data-video="videos/portfolio/<name>-720.mp4"`
plus `data-video-hd="…-1440.mp4"`; main.js picks the tier by physical pixels
(`devicePixelRatio ≥ 1.5` or `innerWidth × devicePixelRatio > 1440` → HD) and
lazy-injects a muted looping `<video>` when the tile scrolls into view
(pauses off-screen, skipped entirely for reduced-motion/Save-Data/no-JS — the
`<img>` always remains as poster and fallback). A tile without `data-video`
simply stays static (hotel-guest-bedroom is deliberately static: it's a
two-panel diptych and video models pan between the panels instead of holding).
The pre-tier single-resolution clips (`res-cinema-room.mp4` etc.) are kept for
one deploy cycle so Pages' ~10 min stale-HTML window can't 404, then can be
deleted.

**Every new portfolio image added to the grid should get a matching clip.**
Recipe: Seedance 2.0 image-to-video on fal.ai (`~/.claude/skills/media-gen/`,
key in `~/.fal_key` — never in this public repo), 720p, 4–5s, audio off,
`end_image_url` = the source photo so the clip returns to frame 1 and loops
seamlessly. Prompt style: slow gimbal push-in toward a focal point + one
ambient detail (candle flicker, foliage, drifting clouds), "nothing in the
room changes", ease back to original framing. QA a 5fps contact sheet before
shipping (models invent people/objects). Then Topaz-upscale the raw clip 4×
(fal `fal-ai/topaz/upscale/video`, Proteus, ~$0.33/clip — upload to fal
storage first, data-URIs are rejected as too long), store the master in
`~/studiochenille-masters/4k/`, and cut both tiers with
`tools/encode-video-tiers.sh` (960/720-class at crf 26 and 1920/1440-class,
both `-preset slow -movflags +faststart -an`). When replacing a shipped clip,
always use a new filename (`?v` doesn't protect immutable-cached media).

### Higgsfield / AI asset generation

Video/image generation now runs pay-per-use on **fal.ai** via the `media-gen`
skill (July 2026: Seedance 2.0 i2v $0.30/s std 720p, Kling 3 std $0.084/s;
always state the dollar cost before submitting). The Higgsfield MCP connector
remains available as fallback: there is no unlimited Seedance on the owner's
Higgsfield Plus plan — Seedance 2.0 **bills credits everywhere** (web app and
Claude MCP connector alike, ~4.5 credits/second at 720p std). Preflight with
`get_cost: true` and check `balance` before generating anything.
